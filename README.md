2) How do I approach developing programs?

What new development strategies did you use while working on your 3D scene?

A new development strategy I used was creating a callback function. For example, I created a "glfwSetKeyCallback" function that would toggle between orthorgraphic and perspective projection by pushing the p key. It works by registering the function I created -- PKeyCallback -- with GLFW -- so that it is called whenever the P Key is pressed.
Another new strategy I used was the process above where I loaded meshes, "colored" them, lit the scene, and set up camera controls.

How did iteration factor into your development?

Iteration factored into development by doing the process in phases. The process I described above -- I did those steps in that order. That is, I made sure the meshes loaded and looked right first. Then i made sure the scene was lit correctly. And then after that, I added the camera controls. Each phase/iteration added something to the scene relative to the last.

How has your approach to developing code evolved throughout the milestones, which led you to the project’s completion?

My approach to developing code has evolved by more considering phases or a project. I have done many projects in the past and have finished one phase and gone some percentage through the next -- it perhaps wasn't the best. I think it is cleaner to do one thing at a time and then come back later when a new phase begins. The milestones I worked on in this class assisted with that evolution because each milestone was associated with the mesh/texturing/lighting/camera movement phase above.
